## 零、一致性可行性分析

### 0.1 纯文本提示词的硬限制

AI 图像生成器**没有跨次记忆**。每次调用都是独立推理。说"SAME as Master"只是语义提示，不能强制像素级一致。

### 0.2 我们能做到的上限

| 能做到 | 做不到 |
|--------|--------|
| 形状轮廓相似（110%椭圆 vs 95%圆） | 像素级完全一致 |
| 颜色在 HEX ±15 范围内 | 精确色号 |
| 角度一致（俯视10度前倾） | 完全相同的透视 |
| 光源方向一致（左上45度） | 完全相同的高光位置 |
| 品种特征可识别（刺型/叶型） | 同一颗榴莲的"脸" |

### 0.3 本方案的核心策略

**"身份指纹"法**——每颗榴莲定义一个独特的视觉特征组合，所有变体提示词中**完整嵌入**这个指纹：

```
身份指纹 = 品种名 + 个体编号 + 独特标记 + 形状参数 + 颜色色号 + 刺的特征 + 构图锁定
```

变体提示词**不引用**"参见主参考"，而是**完整复述**身份指纹，再追加增量描述。每条提示词自包含，AI 拿到的是完整画像。

### 0.4 如果文字提示达不到要求

生成一张满意的主参考后，**后续变体使用图生图模式**（image-to-image），以主参考作为输入图，配合增量提示词。这是确保一致性的最可靠手段。

### 6.2 逐张验证标准

每生成一张图，对照其品种主参考检查：

| 检查项 | 方法 | 通过标准 |
|--------|------|---------|
| 形状一致 | 两张图叠加对比外轮廓 | 偏差 <5% |
| 身份标记可见 | 肉眼检查 | 三角刺(金枕)/双刺簇(干尧)/冠刺(猫山王)/斑点(开裂) 清晰可见 |
| 角度一致 | 检查透视 | 10度前倾俯视，无侧面/正面混入 |
| 光源方向一致 | 检查高光和阴影 | 左上45度高光、右下阴影 |
| 颜色一致 | 取色器 | 各品种主色 HEX ±15 |
| 序列连续 | CP-01→02→03→04→05 逐帧对比 | 形状无突变，裂缝自然渐进 |

### 6.3 一致性无法达标时的后备方案

如果批次2~8 中某张图与主参考偏差过大（无法通过验证标准），采用以下策略：

**策略 A**：同一提示词重新生成 3 次，选最接近的

**策略 B**：取主参考图（如 D-01v4），使用图生图（image-to-image）模式，以主参考为输入图，以变体提示词为文本条件，强度设为 0.6-0.8。这是确保一致性最可靠的手段。

**策略 C**：如果图生图也不理想，手动用 Photoshop 或在线工具将主参考图调色/加裂纹/加特效，模拟变体效果。
