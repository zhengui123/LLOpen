> 版本：MVP v1.0 | 日期：2026-06-15
> 原则：**先跑通流程，后续可迭代**
> 对比完整版：砍掉加工系统、顾客竞价、神秘榴莲；品种从5减到3；贴图从110减到~25张

## 1. 游戏概述

### 一句话介绍

买榴莲 → 划刀开 → 一口价卖出，循环攒钱升级商店。

### MVP 不做的事情

| 砍掉的功能 | 原因 | 后续迭代 |
|-----------|------|---------|
| 加工系统（8条路线）| 增加开发量 + 增加贴图量 | v1.1 |
| 顾客竞价（7类顾客）| 逻辑复杂，先跑通固定价 | v1.1 |
| 神秘榴莲（盲盒）| 多一种品种 = 多一套贴图 | v1.2 |
| 精细刀（看广告逐房揭示）| 广告位已移除（频率过高）| 考虑中 |
| 离线收益 | 需要后端，MVP先不做 | v1.2 |
| Spine骨骼动画 | 安装配置复杂，先用静态图 | v1.1 |

### MVP 保留的核心

| 保留的功能 | 优先级 | 说明 |
|-----------|--------|------|
| 市场3选1购买 | P0 | 核心交互入口 |
| 外观品质系统（4级）| P0 | 核心博弈点 |
| 权重偏移概率叠加 | P0 | 外观影响出肉率的核心算法 |
| 划刀开榴莲交互 | P0 | 核心情绪爆点 |
| 出肉率揭示 + 估价 | P0 | 游戏反馈核心 |
| 固定价售卖 | P0 | 简化的经济闭环 |
| 5个核心广告位 | P0 | 商业化基础 |
| 2级商店升级 | P1 | 基础进度系统 |

## 2. 核心玩法循环（MVP简化版）

```
┌──────────────────────────────────────────┐
│                                          │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐ │
│  │ 买榴莲    │──→│ 开榴莲    │──→│  售卖     │ │
│  │ 3选1+外观 │   │ 划刀揭示  │   │ 固定价    │ │
│  └──────────┘   └──────────┘   └──────────┘ │
│       ↑                            │        │
│       └──────── 收入反馈 ←─────────┘        │
│                                          │
│  驱动循环：赚的钱 → 买更好的榴莲 → 赚更多的钱   │
└──────────────────────────────────────────┘
```

**单局流程**：看余额 → 进市场选品种 → 3选1挑外观 → 划刀开榴莲 → 揭示出肉率 → 固定价卖出 → 重复

## 3. MVP 玩法设计

### 3.1 买榴莲（选购阶段）

#### 场景：榴莲批发市场

玩家进入市场，**先选择品种**（金枕/干尧/猫山王），然后市场展示**3颗该品种榴莲**供挑选。每颗榴莲有**不同的外观品质**。

#### 外观品质系统

| 外观等级 | 视觉特征 | 价格系数 | 出现概率 | 概率偏向 |
|---------|---------|---------|---------|---------|
| 劣质外表 | 壳有裂纹、叶枯萎 | ×0.8 | 35% | 偏向空壳/少肉 |
| 普通外表 | 标准外观 | ×1.0 | 40% | 无偏向 |
| 优质外表 | 壳色鲜亮、叶新鲜 | ×1.5 | 20% | 偏向正常/满肉 |
| 极品外表 | 金光笼罩、叶翠绿 | ×2.2 | 5% | 强烈偏向满肉/爆肉 |

#### 外观与品种的概率叠加（权重偏移法）

```
最终概率权重 = 品种基础权重 + 外观偏移权重
最终概率 = 归一化(最终概率权重)
```

**外观偏移权重表**（应用于5个出肉率档位）：

| 外观等级 | 空壳 | 少肉 | 正常 | 满肉 | 爆肉 | 偏移说明 |
|---------|------|------|------|------|------|---------|
| 劣质外表 | +15 | +10 | -10 | -10 | -5 | 坏结果概率提升 |
| 普通外表 | 0 | 0 | 0 | 0 | 0 | 无偏移 |
| 优质外表 | -10 | -5 | +10 | +5 | 0 | 好结果概率提升 |
| 极品外表 | -15 | -10 | +5 | +10 | +5 | 极品结果概率提升 |

#### 市场上3颗榴莲生成规则

| 规则项 | 说明 |
|--------|------|
| 品种统一 | 3颗榴莲品种相同 |
| 外观随机 | 3颗外观等级独立随机 |
| 价格差异 | 同品种不同外观价格不同（×外观价格系数）|
| 购买价格 | 品种基础价 × 外观价格系数 |

#### 榴莲品种（MVP仅3种）

| 品种 | 价格(元) | 基础出肉率 | 房数范围 | 基础权重 [空壳,少肉,正常,满肉,爆肉] |
|------|---------|-----------|---------|------|
| 金枕 | 80 | 30% | 3-5房 | [30, 35, 25, 8, 2] |
| 干尧 | 200 | 35% | 3-6房 | [20, 30, 25, 18, 7] |
| 猫山王 | 400 | 42% | 4-7房 | [10, 20, 25, 30, 15] |

### 3.2 开榴莲（揭示阶段）

#### 简化的交互设计（MVP版）

**普通刀模式（一刀切开）**：
- 玩家在榴莲顶部**滑动划线**
- 屏幕出现**裂缝动画 + 音效** + 轻微震屏
- **全部房数一次性揭示**：显示每房的果肉状态
- 计算总结：出肉率 + 最终估价

**出肉率评级**：

| 出肉率 | 评级 | 视觉反馈 |
|--------|------|---------|
| 0%-15% | 空壳 | 灰色 + 碎裂 |
| 15%-30% | 小亏 | 黄色 + 苦笑 |
| 30%-45% | 回本 | 蓝色 + 平淡 |
| 45%-60% | 小赚 | 绿色 + 微笑 |
| 60%-75% | 大赚 | 金色 + 大笑 |
| 75%+ | 榴莲之王 | 彩虹 + 全屏特效 |

#### 出肉率算法（MVP简化版）

```
最终出肉率 = 品种基础值 + 随机波动(-15%~+20%)
```

说明：MVP版不开启"运气系数"，但外观系统的权重偏移在品种基础权重层面生效，影响出肉率档次而非数值本身。

### 3.3 售卖（固定价阶段）

MVP版售卖**不使用顾客竞价**，改为固定价格回收：

```
卖出价格 = 果肉固定单价 × 出肉率
```

| 果肉品质 | 固定单价(元) |
|---------|-------------|
| 正常果肉 | +50 |
| 满肉果肉 | +100 |
| 爆肉果肉 | +250 |
| 榴莲之王 | +500 |

> 这是MVP最简化的处理方式。完整版的顾客竞价系统（7类顾客竞价博弈）将在v1.1实现。

## 4. MVP 经济系统

### 4.1 收入来源

| 来源 | 说明 |
|------|------|
| 售卖榴莲 | 固定价格回收（见3.3）|
| 看广告奖励 | 5个广告位（见5.1）|

### 4.2 支出

| 支出 | 说明 |
|------|------|
| 购买榴莲 | 品种价格 × 外观价格系数 |
| 商店升级 | 2级升级（见4.3）|
| 去广告卡 | ¥6（首充）|

### 4.3 商店系统（仅2级）

| 等级 | 费用(元) | 效果 |
|------|---------|------|
| Lv.1（初始）| 0 | 基础回收价 |
| Lv.2 | 500 | 回收价 + 20% |

> 完整版有5级商店升级，MVP仅2级。

## 5. MVP 广告系统

### 5.1 5个核心广告位

| 编号 | 广告位 | 触发时机 | 展示方式 | 奖励 | 每日上限 |
|------|--------|---------|---------|------|---------|
| 1 | 免费试闻 | 购买前（选择榴莲时）| 按钮 | 获得该榴莲的微弱线索 | 无 |
| 2 | 加工加速 | （MVP无加工，暂留位置）| - | - | - |
| 3 | 顾客加价 | 售卖时 | 按钮 | 售价 + 20% | 10次 |
| 4 | 每日看广告 | 每日首次进入 | 自动 | 随机Buff | 1次 |
| 5 | 复活（重开）| 开出空壳后 | 弹窗 | 免费重开一颗 | 3次 |

> "加工加速"广告位在MVP中保留位置但不实现，后续加工系统上线后启用。

### 5.2 广告频率控制

| 控制项 | MVP数值 |
|---------|---------|
| 每日总上限 | 10次（MVP简化） |
| 复活每日上限 | 3次 |
| 顾客加价每日上限 | 5次 |

## 6. MVP 数据模型

### 6.1 DurianData

```csharp
public struct DurianData
{
    public int id;
    public VarietyType variety;        // 金枕/干尧/猫山王
    public AppearanceType appearance;  // 劣质/普通/优质/极品
    public float appearancePriceMultiplier; // 外观价格系数
    public int basePrice;              // 品种基础价格
    public float yieldRate;            // 出肉率（生成时计算）
    public int roomCount;              // 房数
    public bool[] roomResults;         // 每房结果（true=有肉）
}
```

### 6.2 AppearanceType

```csharp
public enum AppearanceType
{
    Poor = 0,    // 劣质
    Normal = 1,  // 普通
    Good = 2,    // 优质
    Premium = 3  // 极品
}
```

### 6.3 VarietyType（MVP仅3种）

```csharp
public enum VarietyType
{
    JinZheng = 0,   // 金枕
    GanYao = 1,     // 干尧
    MaoShanWang = 2 // 猫山王
}
```
