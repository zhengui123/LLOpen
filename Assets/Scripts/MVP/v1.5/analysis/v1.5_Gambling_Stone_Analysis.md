# v1.5 —— 赌石复刻分析与精简调整

> 日期：2026-06-26
> 参考文章：游戏茶馆《"赌石"独游意外出圈后，同类上了畅销榜TOP10》
> 调整原则：砍冗余、加爆点、流程最简

---

## 一、赌石游戏的爆点拆解

### 核心公式

```
赌石爽感 = 渐进揭示 × 实时估值波动 × 随时可卖的紧张感
```

文章原文：
> "一刀穷，一刀富，一刀穿麻布，一刀住别墅"
> "同一块石头5秒前后估值可能天差地别"
> "只要有买家出价，玩家就可以随时售出"

### 赌石的4个爆点

| 爆点 | 机制 | 玩家心理 |
|------|------|---------|
| **渐进开窗** | 不是一刀全开，是逐块切开，每切一块估值实时跳动 | "要不要继续？万一下一刀是裂纹？" |
| **随时可卖** | 开石过程中任何时刻都可以成交 | "现在500了，卖不卖？还是再切一刀赌800？" |
| **极端反差** | 极品 vs 废品，5秒内估值翻倍或腰斩 | 强烈的情绪过山车 |
| **加工逆袭** | 即使底子不好，巧妙加工也能翻盘 | "我还有机会" |

### 我们榴莲游戏缺失的两个关键点

| 缺失 | 赌石怎么做 | 我们目前怎么做 |
|------|----------|-------------|
| **随时可卖** | 开窗过程中随时有买家出价，随时可停 | 开完→弹出评级→点卖出——无中途选择 |
| **加工策略** | 套型绕过裂纹，是"操作空间最大的环节" | MVP 砍掉了加工——无任何策略干预 |

## 二、当前 v1.5 的冗余诊断

### 对照赌石，逐系统审视

| 系统 | 赌石中有吗 | 判定 | 理由 |
|------|----------|------|------|
| 图鉴收集 | 无（但合理） | ✅ 保留 | 轻量，不干扰核心流程 |
| 成就系统 | 无 | ❌ **砍掉** | 12个成就=12种弹窗+事件监听+存档。图鉴就是天然成就 |
| 连击系统 | 无 | 🔄 **降为纯视觉** | 连击×3.0崩经济。改为"连续好果计数+火焰特效"，不影响售价 |
| 暴击系统 | 无 | ❌ **砍掉** | 外观系统已经做概率偏移了，再加暴击是"叠床架屋" |
| 每日任务 | 无（但有每日压力） | 🔄 **改为每日目标** | 2个随机任务太复杂。改为单一目标"今日赚N金币" |
| 玩家等级 | 无 | ❌ **砍掉** | 商店等级+金币本身就是进度，不需要 XP 叠一层 |

### 砍掉后的爽点去哪了？

砍掉的不是爽点——是把爽点**集中到开果过程本身**：

```
旧方案（分散的爽点）：
成就弹窗（偶尔） + 暴击闪电（偶尔） + 连击×3（偶尔） + 开果结果 + 每日任务
→ 5种弱反馈，用户记不住哪个是核心

新方案（集中的爆点）：
开果过程中：实时估价跳动 + "现在卖还是继续开？"
开果完成后：评级 + 加工选择 + 图鉴解锁
→ 所有情绪集中在20秒的开果窗口内
```

## 四、开果中途卖 —— 核心新增机制

这是本次调整**最重要**的改动。

### 流程设计

```
进入 OpenPage（完整榴莲显示）
    ↓
玩家开始滑动（壳开裂开始）
    ↓  ← 任何时候可出现"卖出"按钮
┌─────────────────────────────┐
│                             │
│  壳正在开裂（CP进度 0.6）     │
│  当前估价：320 金币           │ ← 实时跳动
│                             │
│  [继续开]    [现在卖出 320]   │ ← 随时可选
│                             │
└─────────────────────────────┘
    ↓ 选"继续开"              ↓ 选"现在卖出"
壳继续裂，估价可能↑或↓        立即跳过剩余动画，以当前估价卖出
    ↓
壳完全打开 → 显示评级 → 进入 SellPage
```

### 估价实时计算

```csharp
// 在 DurianOpener 中
private int CalculateCurrentEstimate(DurianData durian, float openProgress)
{
    // openProgress 0~1 映射到已揭示的果肉比例
    float revealedRatio = Mathf.Lerp(0.3f, 1.0f, openProgress * openProgress);
    
    // 根据已揭示的出肉率和当前进度估算价值
    float estimatedYield = Mathf.Lerp(0f, durian.yieldRate, revealedRatio);
    float basePrice = GetYieldGradePrice(EstimateGrade(estimatedYield));
    
    return Mathf.RoundToInt(basePrice * (1f + shopBonus));
}
```

### "现在卖出"按钮的出现时机

- 壳开裂到 30% 以上时出现（此时已能看到部分果肉，有判断依据）
- 估价随开裂进度实时跳动
- 数字变大时变绿、变小时变红（与赌石开窗的实时估值跳动完全一致）

## 五、简化加工 —— 从0回到1

### 设计

开果完成后，不直接跳 SellPage，而是弹出**一个简单的二选一**：

```
┌─────────────────────────────┐
│  出肉率: 52%  |  评级: ⭐⭐⭐  │
│                             │
│  如何取肉？                  │
│                             │
│  [普通取肉]     [精挑细选]    │
│  免费            30金币      │
│  售价: 180 金   售价: 207 金  │
│              (+15%)          │
└─────────────────────────────┘
```

### 规则

| 选项 | 费用 | 效果 | 何时划算 |
|------|------|------|---------|
| 普通取肉 | 0 | 原价 | 出肉率低(<30%)时，+15%也不够cover30金币 |
| 精挑细选 | 30金币 | 售价+15% | 出肉率高(>40%)时，+15%远大于30金币 |

这样玩家需要**快速计算**：这个出肉率值不值得花30金币精挑？——这就是赌石"套型加工"的简化版。

## 六、每日目标 —— 替代每日任务

### 设计

| 旧设计（每日任务） | 新设计（每日目标） |
|------------------|------------------|
| 2个随机任务 | 1个固定目标 |
| 需要点击领取 | 达到自动完成 |
| "买3颗榴莲""开出大赚"等 | "今日赚500金币" |
| 需要每日刷新逻辑 | 单一数值追踪 |

### UI

顶部栏金币旁边显示：
```
💰 1,250   🎯 今日: 320/500
```

达到500时，目标数字金色闪烁 → 自动弹出"今日目标达成! +50金币" → 目标更新为"600"（次日）。

## 七、连击降为纯视觉

### 改动

```csharp
// 旧 ComboSystem → 新 StreakCounter
public class StreakCounter
{
    public int CurrentStreak { get; private set; } = 0;
    
    public void OnDurianOpened(YieldGrade grade)
    {
        if (grade >= YieldGrade.Normal) CurrentStreak++;
        else CurrentStreak = 0;
        
        if (CurrentStreak > PlayerProgression.Instance.BestStreak)
            PlayerProgression.Instance.BestStreak = CurrentStreak;
    }
}
```

- 不影响售价（从 SellManager 中删除 comboBonus）
- 只显示火焰特效（CO-01）+ 数字
- 存档最高连击记录（BestStreak）
