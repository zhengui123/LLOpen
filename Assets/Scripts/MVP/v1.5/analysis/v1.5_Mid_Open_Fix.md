# v1.5 —— 开果中途卖：贴图修正与流程调整

> 日期：2026-06-26
> 问题：v1.2.1 贴图（BO/SH/FL）是为"一口气开完"设计的，不支持中途卖的渐进揭示
> 修正：重写 BO 和 SH 的提示词，更新 Cursor 代码，共影响3张贴图

---

## 一、根因分析

### 问题1：BO 自带壳边缘 → 出现两套壳

v1.2.1 的 BO-JZ-01 prompt：
> "壳已分离到最外侧（仅保留壳的轮廓边缘）"

**后果**：BO 底层始终显示壳边。当 SH（顶层半壳）部分打开时，玩家同时看到：
- SH 的壳边（在中间，正在分离）
- BO 的壳边（在最外侧，一直存在）

→ **两套壳同时出现，画面逻辑错误**。

### 问题2：中途卖时，BO暴露过多信息

如果 BO 显示所有5个腔室，壳开到30%时，玩家通过缝隙已经能看到 BO 的全部腔室结构 → 失去了"猜"的悬念。

### 问题3：SH 宽度不够覆盖 BO 的腔室区域

SH-JZ-01 宽384。两个合起来768。但 BO-JZ-01 也是768，包含壳边缘后，实际腔室区域约500px宽。SH 关闭时刚好覆盖。但如果 BO 去掉壳边后腔室扩大到600px，SH 就盖不住了。

---

## 二、修正方案

### 核心改动：重新定义三层分工

```
修正前（v1.2.1）：
L1 BO: 壳边缘 + 空腔结构 + 壳边缘    ← 壳边和腔在一起
L2 FL: 果肉块
L3 SH: 半壳

修正后（v1.5）：
L1 BO: 仅5个空腔结构（无壳边）       ← 纯腔室
L2 FL: 果肉块（不变）
L3 SH: 半壳（壳边由SH独占）          ← 壳边的唯一来源
```

### BO-JZ-01 重写

**旧设计**：壳分离到最外侧 + 5个腔室 + 壳边缘 → 768×768
**新设计**：仅5个暗色空腔，无壳边 → **600×600**（缩小到只含腔室区域）

这样 SH（合起来768宽）**完全覆盖** BO（600宽），不会出现两套壳。

### SH-JZ-01 调整

**保持不变**：384×768，半壳。但 prompt 补充：壳关闭时左右壳交叠约2mm（模拟真实果壳微咬合），防止中间出现缝隙透出 BO。

## 五、中途卖的渐进揭示流程

```
初始状态：SH 关闭（左右壳微咬合）→ 画面只看到完整壳
  ↓
玩家滑动 20%：SH 分开 20% → 裂缝出现 → BO 的暗色内壁可见
  估价：基于可见腔室数推测 → 显示极低估价
  ↓
玩家滑动 40%：SH 分开 40% → 1-2个腔室可见 → 
  - 如果腔室有肉（FL设定的）→ 估价↑
  - 如果腔室空的 → 估价↓
  ↓
[见好就收] ← 玩家在此处决策
  ↓
玩家滑动 60%：SH 分开 60% → 3-4个腔室可见 → 估价大幅波动
  ↓
玩家滑动 80%+：大部分腔室可见 → 估价趋近真实值
  ↓
100%：SH 完全弹开 → BO+FL 全显 → 评级弹出
```

**估价公式**：
```csharp
private int CalculateCurrentEstimate(DurianData durian, float progress)
{
    // progress 0~1: 壳打开比例
    // visibleRatio: 已揭露的腔室比例（不是线性——前30%慢，后70%快）
    float visibleRatio = Mathf.Pow(progress, 1.5f);
    
    // 基于可见比例和真实出肉率估算
    float estimatedYield = durian.yieldRate * visibleRatio;
    
    // 加入不确定性——赌石的核心张力
    float uncertainty = Random.Range(-0.15f, 0.15f);
    if (progress < 0.6f) uncertainty *= 2f; // 早期不确定性更大
    
    estimatedYield = Mathf.Clamp01(estimatedYield * (1f + uncertainty));
    
    float basePrice = GetYieldGradePrice(EstimateGrade(estimatedYield));
    float shopBonus = shopManager.GetSellBonus();
    return Mathf.RoundToInt(basePrice * (1f + shopBonus));
}
```

**关键**：`Mathf.Pow(progress, 1.5f)` 使得前30%进度只揭示约16%腔室→估价低→诱惑"再开一点"。不确定性在 progress<60% 时加倍→数字剧烈跳动→制造紧张。
