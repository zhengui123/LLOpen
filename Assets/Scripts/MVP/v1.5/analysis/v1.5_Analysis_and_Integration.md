## 一、v1.3 合理性逐项分析

### 1.1 图鉴收集系统 ✅ 保留，小幅调整

| 维度 | 评估 |
|------|------|
| 游戏性 | **高**。每次开果都有"记录价值"，消除空壳的负面感。图鉴驱动玩家主动追求特定结果 |
| 与 v1.2.1 配合 | **好**。图鉴是纯数据层，完全不接触开裂动画。开果→判定出肉率→解锁对应格，逻辑清晰 |
| 潜在问题 | 3品种×6档=18格，但猫山王开出"榴莲之王"的概率极低（基础权重中猫山王压根没有King档） |

**调整建议**：
- 图鉴用 v1.2.1 的5档出肉率（空壳/少肉/正常/满肉/爆肉），把"榴莲之王"合并到"爆肉"档，格子变为 **3品种×5档=15格**，与实际出肉率系统一致
- 或者统一为6档但把"榴莲之王"作为爆肉的高端变体共享同一格

**⏳ 需要你判断**：图鉴是 5档(15格) 还是 6档(18格)？

### 1.3 连击系统 ⚠️ 保留但调数值

| 维度 | 评估 |
|------|------|
| 游戏性 | **高**。连续好运的爽感很强 |
| 经济风险 | **⚠️ 连击×3.0可能崩坏经济**。如果玩家连续10次好果（这在游戏后期概率偏高），每颗售价翻3倍，金币通胀严重 |
| 概率计算 | 金枕"正常以上"概率约60%。10连击概率=0.6^5≈7.8%。5连击需连续5次正常以上，X1.5倍——尚可控。但要到10连击×3.0才崩 |

**调整建议**：
- 连击加成**对半砍**：×1.0→×1.1(3连击)→×1.3(5连击)→×1.5(7连击)→×2.0(10连击)，删除×3.0
- 或者连击只影响"金币奖励加成"而不是售价倍数——连击时额外掉落金币粒子，不影响经济体系

**⏳ 需要你判断**：连击是影响售价（经济系统）还是影响额外金币掉落（独立奖励）？

### 1.4 幸运暴击系统 ✅ 保留

| 维度 | 评估 |
|------|------|
| 游戏性 | **非常高**。"绝地翻盘"是让人分享截图的设计。劣质外表+暴击=爆肉，反差极强 |
| 与 v1.2.1 配合 | **完美**。暴击触发后，系统将果肉层从低版本(FL-JZ-20)切换到高版本(FL-JZ-80或100)。开裂动画自然展示"意外的满肉"，视觉冲击力正好在你三层系统中实现——壳打开→居然这么多肉！ |

**无需调整**。唯一建议：暴击触发时机从"生成榴莲时"改为"壳开始打开的那一刻"。这样玩家的预期是"看看这次运气怎么样"，打开瞬间闪电特效+满肉出现，惊喜感最大化。

**⏳ 需要你判断**：暴击在**购买时就确定**(生成榴莲时判定)还是**开果时才判定**(划刀瞬间)？

---

### 1.5 每日任务系统 ✅ 保留但减量

| 维度 | 评估 |
|------|------|
| 游戏性 | **中**。轻量微信小游戏标配，但不能喧宾夺主 |
| 潜在问题 | "看1次广告"任务可能让玩家感觉被逼看广告，降低好感。任务奖励总金币偏高(30~150)，3个任务+全部完成额外奖励≈300金币/天，相当于2-3颗榴莲，可能太快 |

**调整建议**：
- 删除"看广告"任务类型——广告应该由玩家有需求时主动触发，不该被任务驱动
- 改为"签到式"——首次登录自动领取20金币，不需要看广告
- 每日2个任务（非3个），减少压力

## 二、v1.4 合理性分析

### 2.1 v1.4 与 v1.2.1 的冲突点

v1.4 定义了三套榴莲贴图：
- D 系列（完整榴莲，外观变体，已开状态）：用于 MarketPage 展示 + OpenPage 静态图
- CP 系列（开裂阶段序列）：**已被 v1.2.1 废弃**，替换为 BO/FL/SH 三层系统

v1.2.1 新增了三层系统：
- BO 系列（打开底图）
- FL 系列（果肉层，5版本）
- SH 系列（半壳）

**关键问题**：D-O 系列（"已开空壳/爆肉"）和 BO+FL 系列（"打开底图+果肉层"）是**两套不同的"打开后"图像**，功能重叠。

| 对比 | D-O 系列（v1.4） | BO+FL 系列（v1.2.1） |
|------|-----------------|---------------------|
| 用途 | 静态展示 | 动画分层 |
| 张数 | 3品种×2极端 = 6张 | 3品种×(1底+5肉+1壳) = 21张 |
| 画布 | 512×512 | 768×768 |
| 视觉风格 | 完整画面 | 分层组件 |

### 2.2 建议：明确分工，消除重叠

```
D 系列（v1.4）→ 仅用于 MarketPage 静态展示 + 背包卡片
BO+FL+SH 系列（v1.2.1）→ 仅用于 OpenPage 开裂动画

两套贴图不互串。MarketPage 不需要 BO，OpenPage 不需要 D-O。
```

**调整**：
- v1.4 的 D-O 系列保留，但定位改为"MarketPage 上展示的已开效果预览图"和"背包中榴莲卡片的结果展示图"
- v1.2.1 的 BO+FL+SH 保持为开裂动画专用
- **D-O 的身份指纹标记必须与 BO/SH 一致**——同一颗"GoldenPillow-01"的三角刺同时出现在 D-O 和 BO-JZ-01

**具体操作**：v1.4 的 D-O 系列提示词已经包含身份指纹，v1.2.1 的 BO-JZ-01 提示词也已包含同一指纹。确保两边用同一身份标记，AI 生成的完整图和分层图就能保持视觉一致性。

### 2.3 v1.4 的 CP 系列清理

CP-01~05 在 v1.2.1 已被废弃，需要在 v1.4 文档中加一个醒目的弃用声明：

> **⚠️ CP-01~CP-05 已在 v1.2.1 废弃。开裂动画改为 BO/FL/SH 三层分离方案。如需开裂效果，请使用 v1.2.1 文件夹下的贴图，不再生成 CP 系列。**

### 2.4 v1.4 总体结论

| 部分 | 判定 | 行动 |
|------|------|------|
| D/D-A 系列（完整榴莲） | ✅ 保留 | 不变，用于 MarketPage |
| D-O 系列（已开榴莲） | ⚠️ 保留但缩小用途 | 仅用于静态展示预览，不用在动画中 |
| CP 系列（开裂） | ❌ 废弃 | 标记弃用，不再生成 |
| 身份指纹体系 | ✅ 扩展 | 确保 BO/SH 系列也使用同一指纹 |
