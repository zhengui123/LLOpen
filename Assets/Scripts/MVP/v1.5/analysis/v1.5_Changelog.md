### 附加决策：外观系统改用调色

**旧方案**：每个品种×3种外观变体 = 9张贴图（D-A01~D-A09），AI单独生成

**新方案**：只保留 D-01/D-02/D-03（普通外表）作为基础贴图。在 Unity 中通过修改 SpriteRenderer/Image 的 color 属性实现外观变化：

| 外观等级 | color 值 | 视觉效果 |
|---------|---------|---------|
| 劣质 | (0.75, 0.75, 0.75, 1.0) ≈ 灰色调 | 暗淡、褪色 |
| 普通 | (1.0, 1.0, 1.0, 1.0) = 白色 | 不变，原图 |
| 优质 | (1.1, 1.05, 0.9, 1.0) ≈ 微金 | 明亮、微光泽 |
| 极品 | (1.2, 1.1, 0.7, 1.0) ≈ 浓郁金 | 金色光晕 |

> 如果 color 乘法效果不够好（特别是极品需要"金光笼罩"），改用 Material 属性块 + 简单 Shader（Additive glow），不增加贴图。

### 4.2 新增外观调色方法

```csharp
/// <summary>获取品种基础贴图（仅普通外表）</summary>
public Sprite GetUnopenedSprite(VarietyType variety)
{
    return variety switch
    {
        VarietyType.JinZheng => jinZhengNormal,
        VarietyType.GanYao => ganYaoNormal,
        VarietyType.MaoShanWang => maoShanWangNormal,
        _ => jinZhengNormal
    };
}

/// <summary>获取外观对应的调色 color 值</summary>
public Color GetAppearanceColor(AppearanceType appearance)
{
    return appearance switch
    {
        AppearanceType.Poor => new Color(0.75f, 0.75f, 0.75f, 1f),
        AppearanceType.Normal => Color.white,
        AppearanceType.Good => new Color(1.10f, 1.05f, 0.90f, 1f),
        AppearanceType.Premium => new Color(1.20f, 1.10f, 0.70f, 1f),
        _ => Color.white
    };
}
```

## 五、Cursor 更新 —— Q1：图鉴5档15格

### 5.1 PlayerProgression 更新

```csharp
// 图鉴数组从 18 格缩减为 15 格
public bool[] CollectionUnlocked = new bool[15];    // 3品种×5档
public int[] CollectionBestPrice = new int[15];

// 索引计算（v1.5 修正）
public static int GetCollectionIndex(VarietyType variety, YieldGrade grade)
{
    // 5档制：Empty=0, Low=1, Normal=2, High=3, Perfect=4
    // Perfect 包含 爆肉(60%+) 和 榴莲之王(75%+)
    return (int)variety * 5 + (int)grade;
}

// 集齐判定
public bool IsRowComplete(VarietyType variety)
{
    int baseIdx = (int)variety * 5;
    for (int i = 0; i < 5; i++)
        if (!CollectionUnlocked[baseIdx + i]) return false;
    return true;
}

public int GetCollectionCount()
{
    int count = 0;
    for (int i = 0; i < 15; i++)
        if (CollectionUnlocked[i]) count++;
    return count;
}
```

## 六、Cursor 更新 —— Q3：暴击购买时确定

```csharp
// DurianGeneratorSystem.cs 中，生成榴莲时就判定暴击
public DurianData GenerateMarketDurians(VarietyType variety)
{
    // ... 现有生成逻辑 ...
    
    // Q3 决策：暴击在购买时确定
    var (isCrit, bonus, critType) = criticalSystem.RollCritical(
        variety, appearance, yieldGrade);
    
    if (isCrit)
    {
        durianData.yieldRate = Mathf.Min(1f, durianData.yieldRate + bonus);
        durianData.isCriticalHit = true;   // 标记已暴击
        durianData.critType = critType;
        durianData.yieldGrade = RecalculateGrade(durianData.yieldRate);
        
        // ⚠️ 注意：暴击已在此时确定，开果动画只是"揭晓"而非"触发"
        // 所以 CriticalLuckSystem 不要在开果时重复判定
    }
    
    return durianData;
}

// 在 MarketPage 上给暴击榴莲加微弱提示（可选）
// 如果外观极品+暴击，卡片上显示微小金色光点，暗示"这个可能不错"
```

// SellManager.cs —— Q2 决策：连击影响售价
[Inject] private ComboSystem comboSystem;

public int CalculateSellPrice(DurianData durian)
{
    float basePrice = GetYieldGradePrice(durian.yieldGrade);
    float shopBonus = shopManager.GetSellBonus();
    float comboBonus = comboSystem.GetComboMultiplier(); // v1.3 连击
    return Mathf.RoundToInt(basePrice * (1f + shopBonus) * comboBonus);
}
